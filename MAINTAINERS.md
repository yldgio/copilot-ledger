# Maintainers

Maintainers are responsible for project direction, issue triage, release decisions, security response, and review of proposed changes.

## Current maintainers

- Repository owners

## Maintainer responsibilities

- Keep documentation aligned with implementation.
- Triage issues and label them clearly.
- Confirm scope before accepting pull requests.
- Protect user privacy and avoid requesting raw ledger files unless necessary.
- Coordinate security fixes through the private vulnerability reporting process.

## Decision making

For now, maintainers make decisions by consensus in issues or repository discussions. If consensus is not possible, repository owners make the final decision.
