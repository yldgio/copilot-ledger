# Security Policy

## Supported versions

Security fixes target the current `main` branch until versioned releases are introduced.

## Reporting a vulnerability

Please do not open public issues for vulnerabilities.

Use GitHub private vulnerability reporting from the repository's **Security** tab. If private vulnerability reporting is unavailable, contact a maintainer privately and include only the minimum details needed to start triage.

Include:

- Affected version or commit
- Install location: user extension folder or workspace extension folder
- Impact summary
- Reproduction steps or proof of concept
- Any known mitigations

## Data handling

Do not attach raw `.ledger/*.jsonl` files unless requested. If usage records are needed for triage, redact user identifiers, repository names, timestamps, and any organization-specific details first.

## Response process

Maintainers will triage reports, coordinate a fix when needed, and publish remediation guidance appropriate to the impact.
